# Project: GoReda - On-Chain Escrow Marketplace with Validator Nodes

## 한 줄 요약
스테이킹된 검증자 노드가 물리적 상품을 중간 검수하고, 모든 배송 단계를 온체인에 기록하는 솔라나 P2P 커머스 프로토콜. Solana Frontier Hackathon 데모.

- .env 등 시크릿 파일 절대 커밋 금지
- 언어는 영어로 만들 것

## 🎯 Demo North Star (이게 모든 결정의 기준)
심사위원에게 보여줄 두 개의 핵심 wow 모먼트:

**#wow 1: 실시간 온체인 배송 추적**
- 셀러가 트랜잭션 보내는 순간 → 바이어 화면 타임라인이 자동 갱신
- 각 단계마다 Solscan 링크 + 트랜잭션 해시 노출
- "DB가 아니라 진짜 블록체인입니다" 가 한눈에 증명되어야 함
- 프론트에서 바이어 측에서 구매 request 날릴 순간부터 주문 내역 탭에서 타임라인 볼 수 있어야 함

**#wow 2: 즉시 환불 (Instant Refund)**
- 바이어가 환불 버튼 누르고 1 트랜잭션 안에 자금 회수
- 에스크로 잔액이 시각적으로 0으로 떨어지고 바이어 지갑 잔액이 올라감
- "쿠팡은 3일, 우리는 3초" 메시지가 명확

이 두 개를 빛나게 하지 못하는 기능은 모두 후순위.
배송사는 사용하고 있다고 가정하고 데모 작성. 

### 바이어 입장 시뮬레이션
프론트에는 바이어 입장에서 봤을 때, 상품 고를 수 있어야하고, 상품 구입을 누를 수 있어야 함. 
바이어가 볼 수 있는 배송 프로세스:
  1. 바이어의 구매 요청 
  2. 셀러의 주문 확인 
  3. 셀러 -> 검증자로의 배송 시작 
  4. 셀러 -> 검증자로의 배송 완료 
  5. 검증자 -> 바이어로의 배송 시작 
  6. 바이어의 구매 확정 버튼 클릭

이 순서이고, 환불 시뮬레이션은 셀러가 주문 확인 버튼을 누르고 배송 시작 버튼을 누르기 전에 시뮬레이트 할거야.

### 셀러 입장 시뮬레이션
프론트에서 들어온 주문을 셀러가 검증자에게 넘길 때, 검증자 노드 목록을 선택할 수 있어야 하며, 해당 검증자 노드는 데모에서는 집어넣는 임의 값이기 때문에 한 8개 정도 띄워주고 현재 내 위치로부터 몇 미터 떨어져 있는지 위치 걍 하드코딩해서 나타내주셈(+ 검증자가 스테이킹하고 있는 sol 금액도 각각 하드코딩해 놔). 
그리고 데모에서는 검증 프로세스를 잘했다치고 건너뛰고 온체인 배송 추적 및 에스크로 잘 갔나 확인할 거임.

## 핵심 액터 & 비용
- **Seller**: 배송비 1건 + 검증자 수수료 부담. 상품 등록/주문 확인/검증자 발송.
- **Buyer**: 배송비 1건 + 상품 가격 에스크로에 lock. 실시간 추적/환불/구매 확정.
- **Validator Node**: 100 SOL 스테이킹 시 검증자 노드로 등록 가능 (악의 시 슬래싱). 검수 후 바이어 발송. 검증 수수료 수익.

## State Machine
LISTED → PURCHASED(escrow_locked) → ORDER_CONFIRMED 
  → SHIPPED_TO_VALIDATOR(tx_hash_1 onchain) 
  → VALIDATED | REJECTED 
  → SHIPPED_TO_BUYER(tx_hash_2 onchain) 
  → DELIVERED 
  → COMPLETED (1주일 자동 OR 수동 확정)

## 환불 규칙
- 즉시 환불 가능: PURCHASED ~ SHIPPED_TO_VALIDATOR 직전
- 검증자 거절 → 바이어 즉시 환불 / 셀러는 회수 배송비 추가 지불
- 검증자 슬래싱: 컴플레인 트리거만 데모 (실제 검증 로직 X)

## 데모 가정
- Validator 시드 가정 (이미 스테이킹 완료)
- 배송사 외부 시스템 가정 → 운송번호는 mock 자동 생성
- 상품: 와인 섹션 -> 데모니까 하나의 셀러(내 다른 지갑 주소)가 모든 상품을 올린 방식으로 가정
- Devnet
- 7일 유예기간 → 어차피 데모에서는 시뮬레이션 돌릴 때, 확정 버튼 직접 클릭할 예정

## 기술 스택
- Smart Contract: Anchor (Rust)
- Frontend: Next.js 14 App Router + Tailwind -> https://www.ladygaga.com/us-en/shop/collections/all 이거 사이트 참고해서 이 프로젝트에 맞게 구현. 
- Wallet: @solana/wallet-adapter (Phantom)
- 온체인 동기화: Anchor client + accountSubscribe (WebSocket)

## 코드 작성 원칙
1. **WebSocket 우선**: 폴링은 폴백. 실시간 wow 모먼트가 살아야 함
2. **트랜잭션 해시 노출**: 모든 상태 전이 후 Solscan 링크 자동 표시 
3. **에스크로 모듈 분리**: 자금 흐름 코드는 `escrow.rs`로 격리
4. **상태 머신 단일 진실원**: 프론트의 OrderStatus enum과 컨트랙트 enum 동기화
5. **Happy path 먼저**: 엣지케이스는 데모 후. 발표 시나리오 깨지지 않게.
